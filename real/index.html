Fuzzy Path of Mine

I've been programming on and off since my third year of high-school. Started with Pascal and the thrill that came from seeing the computer do exactly as I tell it to, whether right or wrong. Continued with MATLAB while studying mechanical engineering. Somehow I got tangled up with Linux, CLI and vim, then distro-hopped until Ubuntu settled with me. Picked up touch-typing along the way – my current speed is around 100 WPM. Learned some C for my Master's Thesis that addressed the variants of linear and circular interpolations for real-time control of a plasma cutter. At that time, I was a mechanical engineer with a knack for programming.

During my PhD studies, some projects literally shoved me neck deep into situations that overwhelmed me. But, I managed. It was too much fun to quit. "You know some C? Great, now you'll learn C++! And you'll use OpenGL." – and I just kept going. Learned Python on a hunch, seemed cool enough. Cool indeed – it supercharged my abilities and my selftaught learning pace, since I was freed from the rigor of the C++ compiler. Did some Python projects, learned OOP, Git and some good programming practices after getting clogged in my own spaghetti code, with manual versioning by filename – bet you can imagine how well "main11.2_finalVer_final.py" went. Then got back into C++ and learned it properly, with constness, less segfaults and redness from the compiler.

Currently, I'm grinding LeetCode, sharpening my skills further. I took up Java after C++ became repetitive, in part because Sedgewick's algorithms book is written in it.

Programming Language Omnivore

C/C++ and Python base. Java is most recent, but growing. Lisp/Scheme since SICP is a brilliant book. A touch of C# for Unity games. A smidgen of JavaScript, CSS & HTML because bland presentations lose your audience. And LaTeX for serious texts.
